### (ความรู้เพิ่มเติม) ทำไมต้อง RI Constraints

ทุก foreign key ควรระบุว่าจะทำอะไรเมื่อ parent ถูกลบ/แก้ไข:

| FK | Rule | ความหมาย |
|----|------|----------|
| `customer.country_id` | `ON DELETE RESTRICT` | ลบ country ไม่ได้ถ้ายังมี customer อยู่ |
| `product.units_id` | `ON DELETE RESTRICT` | ลบ unit ไม่ได้ถ้ายังมี product อยู่ |
| `invoice.customer_id` | `ON DELETE RESTRICT` | ลบ customer ไม่ได้ถ้ายังมี invoice อยู่ |
| `invoice.sales_person_id` | `ON DELETE SET NULL` | ลบ sales person ได้ — invoice ยังอยู่ แต่ FK จะเป็น null |
| `invoice_line_item.invoice_id` | `ON DELETE CASCADE` | ลบ invoice แล้ว line items หายตาม |
| `invoice_line_item.product_id` | `ON DELETE RESTRICT` | ลบ product ไม่ได้ถ้ายังมี line item อยู่ |

## ทดสอบครบทุก Flow

| ทดสอบ | วิธี | ผลที่คาดหวัง |
|-------|------|-------------|
| API list | `GET /api/sales-persons` | ได้ 10 แถว |
| API search | `GET /api/sales-persons?search=SP001` | ได้ 1 แถว |
| Invoice list | เปิดหน้า /invoices | เห็น column "Sales Person" |
| สร้าง invoice | กด LoV เลือก sales person | ชื่อขึ้นอัตโนมัติ |
| บันทึกแล้วดู | เปิด invoice ที่สร้าง | เห็น Sales Person บน invoice |
| Print PDF | กด Print PDF | Sales Person ติดมาบน invoice |
| แก้ invoice | กด Edit | sales person โหลดมาจาก DB |
| ไม่ใส่ sales person | สร้าง invoice โดยไม่เลือก | บันทึกได้ปกติ |
